# 配置文件说明

## config.json 配置项

所有应用的行为都可以通过 `config.json` 文件进行自定义。

### 配置项列表

```json
{
  "image_path": "sample.jpg",
  "window_title": "图片查看器",
  "app_name": "图片查看器"
}
```

| 配置项 | 类型 | 必填 | 默认值 | 说明 |
|--------|------|------|--------|------|
| `image_path` | string | 是 | - | 要显示的图片路径（相对路径或绝对路径） |
| `window_title` | string | 否 | "图片查看器" | 窗口标题栏显示的文本 |
| `app_name` | string | 否 | "图片查看器" | **桌面图标显示的名称** |

### 详细说明

#### 1. image_path（图片路径）

指定要全屏显示的图片文件路径。

**支持的格式**：

- 相对路径：相对于可执行文件或配置文件所在目录

  ```json
  "image_path": "sample.jpg"
  "image_path": "images/photo.png"
  ```

- 绝对路径：完整的文件系统路径

  ```json
  "image_path": "/home/user/Pictures/myphoto.jpg"
  "image_path": "C:\\Users\\user\\Pictures\\myphoto.jpg"
  ```

**示例**：

```json
{
  "image_path": "/opt/images/welcome.jpg",
  "window_title": "欢迎界面",
  "app_name": "欢迎程序"
}
```

#### 2. window_title（窗口标题）

设置应用程序窗口的标题栏文本。

**注意**：在全屏模式下，标题栏通常不可见，但在窗口模式或任务栏中会显示。

**示例**：

```json
{
  "image_path": "sample.jpg",
  "window_title": "公司宣传展示",
  "app_name": "宣传展示"
}
```

#### 3. app_name（应用名称）⭐

**这是控制桌面图标显示名称的配置项！**

- 打包脚本会读取此值作为桌面快捷方式的名称
- 如果未设置，会使用 `window_title` 的值
- 如果两者都未设置，使用默认值"图片查看器"

**修改后的效果**：

- 桌面上的图标文字会显示为自定义名称
- 应用程序菜单中显示的名称也会改变

**示例**：

```json
{
  "image_path": "logo.jpg",
  "window_title": "企业Logo展示",
  "app_name": "我的专属应用"
}
```

打包后，桌面上会显示 **"我的专属应用"** 而不是"图片查看器"。

## 修改配置的方法

### 方法一：打包前修改（推荐）

在源码目录直接编辑配置文件：

```bash
nano config.json
# 修改配置...
./build.sh
```

### 方法二：安装后修改

编辑已安装位置的配置文件：

```bash
# 用户级安装
nano ~/.local/share/applications/image-viewer/config.json

# 系统级安装
sudo nano /opt/image-viewer/config.json
```

如果修改了 `app_name`，需要重新创建桌面快捷方式：

```bash
./install-desktop-shortcut.sh
```

### 方法三：直接编辑桌面快捷方式

```bash
nano ~/Desktop/image-viewer.desktop
```

修改 `Name=` 行的值：

```ini
[Desktop Entry]
Name=你想要的新名称
...
```

## 注意事项

1. **JSON 格式要求**：
   - 必须使用双引号包裹键和字符串值
   - 最后一个属性后面不能有逗号
   - 确保文件编码为 UTF-8

2. **路径分隔符**：
   - Linux/麒麟系统使用正斜杠 `/`
   - Windows 使用反斜杠 `\`（需要转义为 `\\`）

3. **权限问题**：
   - 修改系统级安装的配置文件需要 root 权限
   - 使用 `sudo` 命令

4. **生效时机**：
   - 修改 `image_path` 和 `window_title`：重启应用即可生效
   - 修改 `app_name`：需要重新打包或手动更新桌面快捷方式

## 常见问题

**Q: 修改了 app_name 但桌面图标名称没变？**

A: 需要重新运行打包脚本或桌面快捷方式安装脚本：

```bash
./build.sh
# 或
./install-desktop-shortcut.sh
```

**Q: 可以使用中文作为应用名称吗？**

A: 可以！完全支持中文名称，例如：

```json
{
  "app_name": "我的企业应用"
}
```

**Q: 如何恢复默认名称？**

A: 删除 `app_name` 字段，或将其设置为"图片查看器"：

```json
{
  "app_name": "图片查看器"
}
```

# 银河麒麟操作系统 V11 部署指南

## 系统要求

- **操作系统**: 银河麒麟操作系统 V11（基于 Linux 6.6 内核）
- **硬件架构**: 海光 x86_64
- **Python**: Python 3.6+（推荐 Python 3.8+）
- **桌面环境**: UKUI（麒麟桌面环境）或其他支持 .desktop 文件的桌面环境

## 快速开始

### 方法一：一键打包和安装（推荐）

这是最简单的方式，自动完成打包、安装和创建桌面快捷方式。

```bash
# 1. 进入项目目录
cd /path/to/TkinterSample

# 2. 添加执行权限
chmod +x build.sh

# 3. 运行打包脚本
./build.sh
```

脚本会自动完成：
- ✓ 检查并安装依赖（PyInstaller、Pillow）
- ✓ 打包应用为单个可执行文件
- ✓ 安装到 `~/.local/share/applications/image-viewer/`
- ✓ 在桌面创建"图片查看器"快捷方式图标

**完成后**：桌面上会出现"图片查看器"图标，双击即可启动！

---

### 方法二：分步部署

如果需要使用更精细的控制，可以分步执行。

#### 步骤 1: 准备环境

```bash
# 检查 Python 版本
python3 --version

# 如果没有 Python3，安装它
sudo yum install python3        # 对于基于 RPM 的麒麟版本
# 或
sudo apt-get install python3    # 对于基于 DEB 的麒麟版本

# 安装 pip（如果没有）
sudo yum install python3-pip
# 或
sudo apt-get install python3-pip
```

#### 步骤 2: 安装项目依赖

```bash
cd /path/to/TkinterSample
python3 -m pip install -r requirements.txt
```

#### 步骤 3: 打包应用

```bash
# 安装 PyInstaller
python3 -m pip install pyinstaller

# 打包应用
pyinstaller --name="image-viewer" \
    --onefile \
    --windowed \
    --add-data "config.json:." \
    --add-data "sample.jpg:." \
    --clean \
    TkinterSample.py
```

打包完成后，可执行文件位于：`dist/image-viewer`

#### 步骤 4: 安装应用

**选项 A: 用户级安装（无需 root 权限）**

```bash
# 创建应用目录
mkdir -p ~/.local/share/applications/image-viewer

# 复制文件
cp dist/image-viewer ~/.local/share/applications/image-viewer/
cp config.json ~/.local/share/applications/image-viewer/
cp sample.jpg ~/.local/share/applications/image-viewer/

# 添加执行权限
chmod +x ~/.local/share/applications/image-viewer/image-viewer
```

**选项 B: 系统级安装（需要 root 权限，所有用户可用）**

```bash
# 使用提供的安装脚本
chmod +x install-system.sh
sudo ./install-system.sh
```

这会将应用安装到 `/opt/image-viewer/`，并创建系统命令 `image-viewer`。

#### 步骤 5: 创建桌面快捷方式

```bash
# 使用提供的脚本自动创建
chmod +x install-desktop-shortcut.sh
./install-desktop-shortcut.sh
```

或者手动创建：

```bash
# 创建桌面快捷方式文件
cat > ~/Desktop/image-viewer.desktop <<EOF
[Desktop Entry]
Version=1.0
Type=Application
Name=图片查看器
GenericName=Image Viewer
Comment=全屏图片显示应用 - 适用于海光x86架构国产电脑
Exec=$HOME/.local/share/applications/image-viewer/image-viewer
Icon=application-x-executable
Terminal=false
Categories=Graphics;Viewer;
StartupNotify=true
Keywords=image;viewer;fullscreen;
EOF

# 添加执行权限
chmod +x ~/Desktop/image-viewer.desktop
```

## 使用说明

### 启动应用

有三种方式可以启动：

1. **桌面图标**（推荐）：双击桌面上的"图片查看器"图标
2. **应用程序菜单**：在开始菜单/应用菜单中找到"图片查看器"
3. **命令行**：在终端中输入 `image-viewer`（仅系统级安装后）

### 配置应用

编辑配置文件来自定义应用：

```bash
# 用户级安装的配置文件位置
nano ~/.local/share/applications/image-viewer/config.json

# 系统级安装的配置文件位置
sudo nano /opt/image-viewer/config.json

# 或在打包前修改源码目录的配置文件
nano config.json
```

配置项说明：

```json
{
  "image_path": "/home/user/Pictures/myphoto.jpg",
  "window_title": "我的图片查看器",
  "app_name": "自定义应用名称"
}
```

- `image_path`: 要显示的图片路径（支持相对路径和绝对路径）
- `window_title`: 窗口标题
- `app_name`: **桌面图标显示的名称**（重新打包后生效）

支持的图片格式：JPG、PNG、BMP、GIF 等（由 Pillow 库支持）

### 退出应用

按键盘上的 `ESC` 键即可退出全屏显示。

## 高级配置

### 自定义桌面应用名称

如果您希望更改桌面上显示的应用名称，只需修改配置文件中的 `app_name` 字段：

```json
{
  "image_path": "sample.jpg",
  "window_title": "图片查看器",
  "app_name": "我的专属应用"
}
```

然后重新运行打包脚本：

```bash
./build.sh
```

桌面快捷方式会自动使用新的名称。

### 已安装后更改应用名称

如果应用已经安装，可以：

1. 修改安装目录的配置文件：
   ```bash
   nano ~/.local/share/applications/image-viewer/config.json
   ```

2. 重新创建桌面快捷方式：
   ```bash
   ./install-desktop-shortcut.sh
   ```

或者直接编辑桌面快捷方式文件：

```bash
nano ~/Desktop/image-viewer.desktop
```

修改 `Name=` 后面的值即可。

### 自定义应用图标

1. 准备一个 PNG 或 ICO 格式的图标文件（建议尺寸 256x256）

2. 将图标文件复制到应用目录：
   ```bash
   cp my-icon.png ~/.local/share/applications/image-viewer/icon.png
   ```

3. 修改桌面快捷方式文件中的 Icon 字段：
   ```ini
   Icon=/home/用户名/.local/share/applications/image-viewer/icon.png
   ```

4. 刷新桌面或重新登录使更改生效

### 开机自启动

要在登录时自动启动应用：

```bash
# 复制桌面快捷方式到自动启动目录
mkdir -p ~/.config/autostart
cp ~/Desktop/image-viewer.desktop ~/.config/autostart/
```

### 多图片切换（扩展功能）

如需实现多图片轮播，可以修改 `config.json` 添加多个图片路径，然后修改代码实现切换逻辑。

## 故障排除

### 问题 1: 桌面图标不显示或无法点击

**解决方案**：
```bash
# 检查文件权限
ls -l ~/Desktop/image-viewer.desktop

# 确保有执行权限
chmod +x ~/Desktop/image-viewer.desktop

# 检查文件内容是否正确
cat ~/Desktop/image-viewer.desktop

# 确认 Exec 路径是否正确
grep "Exec=" ~/Desktop/image-viewer.desktop
```

### 问题 2: 应用启动后立即退出

**解决方案**：
```bash
# 在终端中运行查看错误信息
~/.local/share/applications/image-viewer/image-viewer

# 或
/opt/image-viewer/image-viewer  # 系统级安装
```

常见原因：
- 缺少依赖库：`python3 -m pip install -r requirements.txt`
- 图片路径错误：检查 `config.json` 中的路径是否正确
- 权限问题：`chmod +x image-viewer`

### 问题 3: 图片不显示或显示空白

**解决方案**：
- 确认图片文件存在且路径正确
- 检查图片格式是否受支持
- 尝试使用绝对路径而非相对路径
- 查看终端输出的错误信息

### 问题 4: PyInstaller 打包失败

**解决方案**：
```bash
# 清理缓存后重试
rm -rf build dist *.spec
pyinstaller --clean --name="image-viewer" \
    --onefile --windowed \
    --add-data "config.json:." \
    --add-data "sample.jpg:." \
    TkinterSample.py

# 如果提示缺少模块，先安装
python3 -m pip install Pillow
```

### 问题 5: 中文显示乱码

**解决方案**：
确保系统安装了中文字体：
```bash
# 对于基于 RPM 的系统
sudo yum install wqy-zenhei-fonts

# 对于基于 DEB 的系统
sudo apt-get install fonts-wqy-zenhei
```

## 卸载应用

### 用户级安装卸载

```bash
# 删除应用文件
rm -rf ~/.local/share/applications/image-viewer

# 删除桌面快捷方式
rm ~/Desktop/image-viewer.desktop

# 删除自动启动项（如果有）
rm ~/.config/autostart/image-viewer.desktop
```

### 系统级安装卸载

```bash
# 删除应用文件
sudo rm -rf /opt/image-viewer

# 删除系统命令链接
sudo rm /usr/local/bin/image-viewer

# 删除系统菜单项
sudo rm /usr/share/applications/image-viewer.desktop
```
